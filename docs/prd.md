# CrossLink — 产品需求文档 v1.0

> 跨端互联工具：手机远程调用家中电脑的 AI 模型，扫码即用，零配置。
> 一次性买断交付，用户数据自主可控。

---

## 1. 产品定位

### 一句话描述

**让手机随时随地使用家中电脑的 AI 算力，安装到扫码不到 30 秒。**

### 核心价值主张

| 用户痛点 | CrossLink 解决 |
|----------|---------------|
| 在外面想用家中 Ollama 模型，但不懂配内网穿透 | 安装即用，内置 P2P 打洞 + 中继 |
| 现有远程工具要配 IP/端口/证书 | 扫码配对，零配置 |
| 云 API 费用高、数据隐私担忧 | 用自家电脑的模型，数据不过第三方 |
| 想给家人/同事用，但配置太复杂 | 授权管理内置，一次安装 |

### 与竞品的区别

| 产品 | 定位 | 差距 |
|------|------|------|
| Tailscale | 通用 VPN，要配 ACL | 太重，非 AI 场景 |
| TeamViewer | 远程桌面 | 只能看画面，不能调用服务 |
| ngrok/frp | 内网穿透，要写配置 | 对普通用户太难 |
| **CrossLink** | AI 服务专用的个人跨端总线 | **开箱即用，专为 AI 场景优化** |

## 2. 目标用户（种子用户画像）

**第一优先级**：在本地跑 Ollama/Llama.cpp 的开发者

- 经常混 GitHub / V2EX / 即刻 / LocalLLaMA 社区
- 家中有一台跑着大模型的电脑（或 NAS/树莓派）
- 有"在外面调用家中模型"的真实需求
- 愿意为便利性一次性付费（59-149 美元）
- 对隐私敏感，不希望数据经过云服务商

**第二优先级**（v1.1+）：有 AI 需求的非开发者

- 被朋友推荐"在家装一个就能手机用 AI"
- 不关心技术细节，只要能聊天就行

## 3. MVP 功能清单

### 3.1 Agent（家中电脑端）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| GUI 安装包 | Windows .exe / macOS .pkg / Linux .deb，一路下一步 | P0 |
| 系统托盘 | 启动后最小化到托盘，显示连接状态图标 | P0 |
| 二维码展示 | 安装完成后自动弹出二维码，供 App 扫描配对 | P0 |
| Ollama 服务发现 | 自动检测本地 Ollama 实例，上报模型列表 | P0 |
| 连接保持 | 与信令服务器维持长连接，心跳上报 | P0 |
| 自动打洞 | 内置 WebRTC，优先直连，失败自动切换中继 | P0 |
| 自动更新 | 检测新版本，后台静默更新 | P1 |
| 开机自启 | 可选开启，托盘设置页配置 | P1 |
| 网络状态指示 | 托盘图标颜色区分：直连(绿) / 中继(黄) / 断开(红) | P1 |

### 3.2 App（手机端）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 扫码配对 | 打开即扫码界面，扫描 Agent 二维码完成绑定 | P0 |
| 设备列表 | 显示已绑定的电脑，在线/离线状态 | P0 |
| 模型列表 | 点击设备后拉取 Ollama 模型列表 | P0 |
| 流式聊天 | 选择模型后进入对话，支持 markdown 渲染 | P0 |
| 对话历史 | 本地保存聊天记录，可删除 | P1 |
| 连接状态 | 顶部状态栏显示直连/中继/断开 | P1 |
| 多设备切换 | 一台手机可绑定多台电脑 | P1 |
| 设置页 | License 信息、服务器地址（未来切换私有 Hub）、深色模式 | P1 |

### 3.3 公共信令/中继服务

| 功能 | 描述 | 优先级 |
|------|------|--------|
| WebSocket 信令 | 交换 WebRTC SDP，建立连接 | P0 |
| STUN/TURN 中继 | 打洞失败时中转数据 | P0 |
| 设备目录 | 维护用户 ID 与在线节点的映射 | P0 |
| 流量控制 | 中继模式下限制带宽，鼓励直连 | P1 |
| 匿名身份 | 基于设备指纹自动生成唯一 ID，无需注册 | P0 |

## 4. MVP 明确不做（v1.1+ 再考虑）

- ❌ 文件浏览 / WebDAV
- ❌ 终端远程访问
- ❌ 多 Agent 节点同时管理
- ❌ 私有 Hub 一键部署（用户仍可手动部署 docker-compose）
- ❌ 子账号 / 家庭共享
- ❌ 权限细分（只读/读写）
- ❌ 自定义角色/权限
- ❌ 非 Ollama 的模型后端（vLLM、llama.cpp server 等）
- ❌ 语音/图片/文件发送
- ❌ macOS 桌面端 App（v1 只有 Agent + 手机 App）
- ❌ iOS 版本（先做 Android，iOS 跟进或同步取决于资源）

> **原则**：宁可砍到骨感，也要保证核心流程的稳定和流畅。

## 5. 交互准则（红线）

1. **安装过程 0 命令行** — 用户从头到尾不用打开终端
2. **连接建立 < 10 秒** — 从扫码到显示模型列表不超过 10 秒
3. **用户无感术语** — 界面中不出现 P2P、TURN、STUN、信令等词
4. **静默降级** — 直连失败自动切换中继，用户只看到"已连接"，不知道背后发生了什么
5. **离线可用** — 已配对的设备若信令服务临时不可用，直连通道不受影响
6. **错误友好** — 任何异常都有中文提示和操作建议，绝不裸奔错误码

## 6. 核心用户流程

### 6.1 首次安装与配对

```
用户下载 Agent 安装包 → 双击 → 下一步 → 完成
                              ↓
Agent 启动，系统托盘出现图标，自动弹出配对二维码
                              ↓
用户打开手机 App → 首次使用直接进入扫码界面
                              ↓
扫码 → App 显示"连接中..." → 3-8 秒后 → "已连接到 我的台式机"
                              ↓
点击设备 → 显示模型列表（llama3:8b, qwen2:7b...）
```

### 6.2 日常使用

```
用户在外面 → 打开手机 App → 自动连接（上次已配对）
                        ↓
选择模型 → 输入问题 → 流式返回 → 就像在用云服务
                        ↓
状态栏显示 🟢 直连 / 🟡 中继 — 用户不用关心，只是告知
```

### 6.3 授权激活

```
用户购买后收到 License Key（邮件/支付页）
→ 在 Agent 托盘菜单点击"输入 License"
→ 粘贴 Key → 激活
→ 离线可用，7 天联网校验一次
→ 更换电脑需联系重置
```

## 7. 成功指标（MVP 发布标准）

| 指标 | 目标值 |
|------|--------|
| 从下载到发出第一条消息的时长 | < 2 分钟 |
| 扫码到连接成功 | < 10 秒 |
| P2P 直连成功率（同城网络） | > 85% |
| 中继模式下首 token 延迟 | < 1 秒 |
| 崩溃率 | < 1% |
| Agent 内存占用（空闲） | < 100 MB |
| Agent CPU 占用（空闲） | < 2% |

## 8. 系统架构概览

```
┌──────────────┐       ┌──────────────────┐       ┌──────────────────┐
│  手机 App     │ ←───→ │  公共信令/中继层   │ ←───→ │  家中电脑 Agent   │
│  (Flutter)   │       │  (Go, pion/webrtc)│       │  (Wails/Go GUI)  │
│              │       │                  │       │                  │
│  - 扫码配对   │       │  - WebSocket 信令 │       │  - 托盘图标       │
│  - 模型列表   │       │  - STUN/TURN 中继│       │  - 二维码展示     │
│  - 流式聊天   │       │  - 设备目录       │       │  - Ollama 代理    │
└──────────────┘       └──────────────────┘       └──────────────────┘
       │                        │                         │
       └──── WebRTC DataChannel (P2P 直连) ──────────────┘
       └──── WebSocket/TURN (中继回退) ──────────────────┘
```

### 数据安全

- P2P 直连：WebRTC 内置 DTLS/SRTP 端到端加密，中间服务器不可见
- 中继模式：DataChannel 本身 E2E 加密，中继服务器只看加密字节流
- 信令服务器：只交换公钥和网络地址信息，不接触内容加密密钥
- Agent 本地存储：License、密钥存在 SQLCipher 加密数据库，主密码存系统密钥链

## 9. 技术栈（已确认）

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent 桌面应用 | Wails (Go + Web 前端) | 跨平台，体积小，有系统托盘 API |
| Agent 核心 | Go + pion/webrtc | 单二进制，并发好，WebRTC 生态成熟 |
| 手机 App | Flutter | 一套代码打 Android + iOS |
| 信令/中继服务 | Go + pion/webrtc | 与 Agent 共享代码和库 |
| 本地数据库 | SQLite (go-sqlcipher) | 零配置，支持加密 |
| 授权签名 | Ed25519 | 短密钥，验签快 |
| 部署（Hub） | Docker + docker-compose | 一键部署（仅高级用户自建用） |
| CI/CD | GitHub Actions | 跨平台编译 Agent + App |

## 10. 里程碑与阶段规划

| 阶段 | 内容 | 预计时长 |
|------|------|----------|
| **P0: 准备** | PRD、POC、依赖验证、CI 搭建 | 1-1.5 周 |
| **P1: 核心连通** | Hub 信令 + Agent 注册 + App 扫码配对 + 中继 ping 通 | 2-4 周 |
| **P2: P2P + Ollama** | WebRTC 直连 + Ollama 代理 + App 聊天 UI | 2-3 周 |
| **P3: 授权 + 部署** | License 系统 + GUI 安装包 + 自动更新 | 1-2 周 |
| **P4: 打磨** | 错误处理、UI 美化、多平台测试 | 2-3 周 |
| **P5: 发行** | 文档、网站、支付对接、种子用户发放 | 持续 |

**MVP 总预计**: 8-14 周（单人或小团队）
